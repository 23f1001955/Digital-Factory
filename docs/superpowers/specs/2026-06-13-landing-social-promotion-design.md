# Landing Page + Social Media Promotion Design

**Date:** 2026-06-13
**Status:** Draft
**Product:** Digital Product Factory

## Overview

Extend the Digital Product Factory pipeline with two optional post-publishing phases: (1) generate and deploy a landing page for the Gumroad product using Google Stitch, and (2) promote the product across social media platforms (Instagram, Threads, Facebook, Pinterest). Both phases are opt-in via wizard prompts and run after Gumroad publishing completes.

## Agent: `landing_agent`

### Responsibilities

1. **Generate product images via Gemini 3.1 Flash** — cover image (always), product mockups (as LLM decides per run)
2. **Write a Google Stitch design prompt** — includes product info, image URLs, Gumroad link, theme preference
3. **Call Stitch API** (`stitch.googleapis.com/mcp`, tool: `generate_screen_from_text`) — receives HTML + screenshot URL
4. **Refine the HTML** via LLM — inject Gumroad "Buy Now" CTA button, fix responsive layout, polish design
5. **Deploy to Vercel** — `POST /v12/deployments` with the HTML file

### Signature

```python
def run(component: ComponentSpec, job_spec: JobSpec, context: dict) -> AgentResult:
    ...
```

Matches the existing agent contract.

### Image Generation (Gemini 3.1 Flash)

- **Always:** 1 cover image (hero/product shot, clean white background, 16:9)
- **As needed (LLM decides per run):** product mockup images, feature screenshots, social media crop variants
- **API:** `google.genai` Python SDK or raw HTTP POST to `https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash:generateContent`
- **Rate limiting:** 60-second delay between multi-image batches
- **Output:** PNG files in `outputs/{slug}/landing/images/`
- **Fallback:** If Gemini key missing, generate placeholder SVG images with product name overlaid

### Stitch Integration

- **API base:** `stitch.googleapis.com/mcp`
- **Auth header:** `X-Goog-Api-Key: <key>`
- **Tool:** `generate_screen_from_text`
- **Input:** A design prompt (generated by LLM) describing the landing page layout, brand colors, sections, and embedded image URLs
- **Output:** JSON with `screenshot_url` (PNG) and `html` (raw HTML string)
- **Expected latency:** 1-3 minutes
- **Fallback:** If Stitch call fails, generate a basic HTML page locally using Jinja2 template `templates/landing/basic.html.j2`

### Stitch Prompt Construction (LLM Phase)

The LLM receives:
- Product name, description, key features (from content_agent output)
- Cover image URL (from Gemini generation)
- Gumroad product URL (from gumroad_publish output)
- Theme selection (from wizard)
- CTA text (from wizard or default "Buy Now on Gumroad")

Output: A plain-text design brief suitable for Stitch's `generate_screen_from_text` tool.

### HTML Refinement (LLM Phase)

The LLM receives the raw Stitch HTML and:
- Injects a Gumroad "Buy Now" button linking to the published product URL
- Ensures responsive / mobile-friendly layout
- Fixes any broken image references or asset paths
- Polishes typography and spacing
- Output: Clean, self-contained HTML file (no external CSS dependencies beyond what Stitch provides)

### Vercel Deploy

- **API:** `POST https://api.vercel.com/v12/deployments`
- **Auth:** `Authorization: Bearer <vercel_token>`
- **Body:** Deployment object with `files` array containing the HTML file and any images (base64 encoded or uploaded separately)
- **Project:** Created on first deploy, subsequent deploys update the same project
- **Output:** `{ "url": "https://<project>.vercel.app", "deployment_id": "..." }`
- **Fallback:** Save HTML locally at `outputs/{slug}/landing/index.html` so user can deploy manually

## Agent: `social_agent`

### Responsibilities

1. **Generate social media images** via Gemini 3.1 Flash — per-platform optimized sizes
2. **Generate per-platform copy** via LLM — captions, hashtags, post text
3. **Post to Instagram / Threads / Facebook** via Facebook Graph API
4. **Post to Pinterest** via Pinterest REST API

### Signature

```python
def run(component: ComponentSpec, job_spec: JobSpec, context: dict) -> AgentResult:
    ...
```

### Image Generation

LLM decides per-platform image needs:
- **Instagram:** 1:1 square (1080x1080) — product hero or mockup
- **Facebook:** 1.91:1 landscape (1200x630) — same as IG with wider crop
- **Pinterest:** 2:3 vertical (1000x1500) — tall pin-style image
- **Threads:** 1:1 or 9:16 — same as IG or portrait

Gemini 3.1 Flash generates all variants in one call batch.

### Copy Generation (LLM Phase)

The LLM receives product info and generates per-platform copy:
- **Instagram:** 2-3 sentence caption, 8-12 hashtags, 1 CTA
- **Threads:** 1-2 sentence thought-starter, 3-5 hashtags
- **Facebook:** Paragraph-length post, 5-8 hashtags, link preview
- **Pinterest:** SEO-optimized pin title, 200-300 char description, 3-5 hashtags

### Facebook Graph API Integration

- **Endpoint:** `https://graph.facebook.com/v21.0/{page_id}/photos` (image post) or `/feed` (link post)
- **Auth:** `Page Access Token` from `FACEBOOK_PAGE_TOKEN`
- **Platforms covered:** Facebook Page, Instagram (via `/{ig_user_id}/media`), Threads (via `/{threads_user_id}/threads`)
- **Note:** Instagram and Threads posting require the Facebook Page to be connected to those accounts in Meta Business Suite

### Pinterest API Integration

- **Endpoint:** `https://api.pinterest.com/v5/pins`
- **Auth:** `Bearer <pinterest_token>`
- **Board:** User-specified board ID from env `PINTEREST_BOARD_ID` (optional — uses first board if not set)

### Graceful Degradation

Each platform is independently wrapped in try/except:
- If Instagram fails (no IG connected to page), Facebook and Pinterest still post
- If Pinterest fails, social captures the error and continues
- If no images available, text-only posts are used
- If social fails entirely, pipeline records `status: "failed"` with error message but does NOT block the rest

## Constraints

1. **No hardcoded secrets** — all API keys from `.env` via `python-dotenv`
2. **Graceful degradation** — each external call wrapped in try/except, falling back to local saves / partial posting
3. **Rate limiting** — Gemini 3.1 Flash has rate limits; implement 60s delay between multi-image batches
4. **Stitch API latency** — `generate_screen_from_text` takes 1-3 minutes; log progress to stderr so pipeline doesn't appear frozen
5. **Resumability** — `landing_page` and `social_promotion` tracked in `job_state.json`; if already `"done"`, skip on resume
6. **Backward compatible** — existing pipelines without landing/social enabled continue to work unchanged
7. **No frontend framework** — Stitch generates raw HTML; Vercel deploy is single-page HTML, no React/Vue/Next
8. **PDF redesign is separate** — the existing PDF formatting issues (whitespace, text size) are not addressed by this spec
